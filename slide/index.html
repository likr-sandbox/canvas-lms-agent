System Overview

The Canvas LMS Agent is an autonomous assistant enabling direct natural language interactions with the Canvas Learning Management System.

  • Dual Interface: Supports a CLI script for fast verification and a React web wrapper dashboard.
  • Smart Orchestration: Leverages the Google Antigravity SDK to configure and manage agent sessions safely.
  • Full Capabilities: Covers course listings, grading submissions, assignments, discussion moderation, and calendar planning.

Architecture Components

  • React + Vite UI: A sleek, responsive dashboard providing real-time chat and tool-tracking logs.
  • FastAPI Backend Wrapper: Integrates the Antigravity Agent runtime and exposes session APIs.
  • Node.js MCP Server: Connects to Canvas REST APIs and exposes tools dynamically.

Canvas LMS MCP Server

Located in canvas-lms-mcp/, this Node.js server implements the Model Context Protocol (MCP) to bridge LLMs with Canvas REST APIs.

Core Features

  • Course & Sections: Inspect course and user section details.
  • Grading: Execute role-adaptive grading workflows.
  • Calendar: Access planners, calendar events, and deadlines.

Antigravity SDK Integration

The Python backend orchestrates the agent using the Google Antigravity SDK.

  • LocalAgentConfig: Configures workspace roots, security policies, skills, and MCP servers.
  • McpStdioServer: Spawns and communicates with the Node.js MCP server over standard input/output.
  • Security Governance: Leverages strict policies (`policy.allow_all()`) to authorize tool execution.

Execution Hooks & Database Logs

Hooks dynamically track tool execution states to the database:

  • pre_tool_call_decide: Logs incoming tool calls with a status of "running".
  • post_tool_call: Updates the database record with completion status, execution time, and any error outcomes.

Web Dashboard Wrapper

The React + Vite application (agent-ui/) provides a premium interface for interacting with the Canvas LMS Agent.

  • Real-time Chat client: Initiates asynchronous agent chat sessions with uvicorn FastAPI backend.
  • Live Tool Tracking: Side panel logs tools running in the background.
  • Local Memos: Exposes database-backed tools (`write_memo`, `read_memo`, `list_memos`) for taking notes.

Local Stack & Development

  • Vite Dev Server: Runs on port 5173 for the frontend.
  • FastAPI API: Runs on port 8000 for backend routing.
  • Persistent Database: SQL database for logs and memos.
